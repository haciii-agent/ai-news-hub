# AI News Hub 迭代路线图 📰

> 一步一个脚印，每版本聚焦 1-2 个核心目标。

---

## 当前版本：v0.4.0 ✅（已完成）

- 亮/暗主题切换（localStorage 持久化）
- 摘要扩充（2000 字符，优先取 content 字段）
- 图片采集与展示（og:image、media:content、页面内 img）
- 14 个数据源（12 RSS + 2 HTML）

---

## v0.5.0 — 稳定与可靠性

**目标：让服务跑得稳、跑得久。**

### 后端
- [ ] **systemd 部署支持**：生成 systemd service 文件，支持开机自启 + 崩溃重启
- [ ] **内置定时采集调度器**：在 Go 内部实现 cron（用 `robfig/cron`），不再依赖外部 cron
- [ ] **采集失败重试策略优化**：指数退避重试（当前只重试 1 次），单个源失败不影响整体
- [ ] **采集历史 API**：`GET /api/v1/collect/history?page=1` 查看历史采集记录
- [ ] **健康检查增强**：`/health` 加入数据库连通性检测、上次采集时间、运行时长

### 前端
- [ ] **错误源提示**：在采集状态 badge 中 hover 显示哪些源失败了
- [ ] **数据源管理页面**：`/sources.html` 展示所有数据源状态（active/failed/unknown）
- [ ] **页面加载性能优化**：减少骨架屏渲染时间，API 响应缓存

### 配置
- [ ] `config.yaml` 增加调度配置段 `scheduler.cron`（默认 `"0 */4 * * *"` 每 4 小时采集）

---

## v0.6.0 — 搜索与发现

**目标：用户能快速找到感兴趣的内容。**

### 搜索
- [ ] **全文搜索**：基于 SQLite FTS5 实现标题 + 摘要全文搜索（移除 MVP 限制）
- [ ] **搜索 API**：`GET /api/v1/articles?search=关键词` 正式可用
- [ ] **前端搜索框**：顶栏增加搜索输入框，实时搜索（debounce 300ms）
- [ ] **搜索结果高亮**：搜索关键词在标题和摘要中高亮显示

### 内容发现
- [ ] **热门标签/关键词**：从标题中自动提取高频词，展示 Top 20 热门标签
- [ ] **时间范围筛选**：增加 "今天 / 本周 / 本月" 快速筛选按钮
- [ ] **相关文章推荐**：文章详情页底部显示 "相关文章"（同分类 + 关键词匹配）

### 后端
- [ ] **FTS5 虚拟表**：`articles_fts` 表，写入时同步更新索引
- [ ] **关键词提取**：简单的 TF-IDF 或基于停用词的中文分词提取

---

## v0.7.0 — 内容深度增强

**目标：从"标题聚合"进化到"内容聚合"。**

### 原文抓取
- [ ] **全文抓取引擎**：采集到文章 URL 后，异步抓取原文正文（Readability 算法）
- [ ] **正文解析器**：使用 `go-readability` 或自实现提取 `<article>` / `<main>` 内正文
- [ ] **正文存储**：`content_html` 字段正式写入（当前已预留但未使用）
- [ ] **异步抓取队列**：采集完标题后，后台 goroutine 队列逐篇抓取正文（限速防封）

### AI 摘要
- [ ] **LLM 摘要生成**：调用外部 LLM API（OpenAI/通义千问/DeepSeek）对原文生成结构化摘要
- [ ] **摘要格式**：标题 + 3-5 条要点（bullet points）+ 一句话总结
- [ ] **摘要缓存**：已生成的摘要存库，不重复调用
- [ ] **配置化**：`config.yaml` 中 `ai` 段配置 API key、模型、提示词模板

### 前端
- [ ] **详情页正文展示**：文章详情页直接展示抓取到的正文（不需要跳转原文）
- [ ] **AI 摘要卡片**：列表卡片中优先展示 AI 生成的要点摘要
- [ ] **原文跳转**：详情页底部保留"阅读原文"按钮

---

## v0.8.0 — 多语言与国际化

**目标：中英文双语体验。**

### i18n
- [ ] **前端 i18n 框架**：提取所有硬编码中文文案到语言包（zh/en）
- [ ] **语言切换**：根据浏览器语言自动选择，支持手动切换
- [ ] **日期时间本地化**：英文环境显示 "2 hours ago"，中文环境显示 "2 小时前"

### 内容翻译
- [ ] **自动翻译**：对中文文章生成英文摘要，对英文文章生成中文摘要
- [ ] **双语标题**：详情页同时展示原文标题和翻译标题
- [ ] **翻译缓存**：翻译结果存库，避免重复翻译

### 配置
- [ ] `config.yaml` 增加 `ai.translate_api` 配置段

---

## v0.9.0 — 通知与推送

**目标：用户不主动看也能收到重要资讯。**

### 推送渠道
- [ ] **DingTalk Webhook 推送**：每日摘要推送到钉钉群（Markdown 格式）
- [ ] **邮件推送**：每日精选 Top 10 推送到订阅邮箱
- [ ] **Telegram Bot**：通过 Telegram Bot 推送热点新闻

### 推送策略
- [ ] **推送规则引擎**：可配置"当某分类出现 N 篇新文章时推送"
- [ ] **去重推送**：同一新闻只推一次
- [ ] **推送历史**：记录已推送的文章 ID，防止重复

### 每日报告
- [ ] **自动生成日报**：每天定时生成 AI 日报（分类汇总 + 热点 + 数据统计）
- [ ] **日报模板**：可自定义日报格式和推送渠道

---

## v1.0.0 — 正式版

**目标：生产就绪，完整产品。**

### 安全
- [ ] **API 认证**：采集 API（POST）增加 token/API key 验证
- [ ] **HTTPS 支持**：内置 TLS 证书或反代配置指南
- [ ] **Rate Limiting**：API 限流（基于令牌桶）
- [ ] **输入验证加固**：所有 API 参数严格校验

### 运维
- [ ] **日志结构化**：JSON 格式日志，支持 `LOG_LEVEL` 环境变量
- [ ] **Prometheus 指标**：`/metrics` 端点暴露采集数、错误数、延迟等
- [ ] **配置热更新**：修改 `config.yaml` 后自动检测并重载（不重启）
- [ ] **数据库备份**：定期自动备份 SQLite 到指定目录

### 用户体验
- [ ] **用户系统（可选）**：注册/登录、个人收藏、自定义数据源
- [ ] **PWA 支持**：离线阅读、添加到桌面
- [ ] **RSS 输出**：提供 `/feed` 端点，让用户自己订阅聚合后的 RSS
- [ ] **OPML 导入导出**：数据源列表支持 OPML 格式

### 前端
- [ ] **响应式全面优化**：平板和手机端完美适配
- [ ] **键盘快捷键**：`j/k` 上下浏览、`o` 打开详情、`/` 搜索
- [ ] **无障碍（a11y）**：ARIA 标签、键盘导航、屏幕阅读器支持
